| Step | Status | Notes |
|------|--------|-------|
| Landing page (`/`) | ✅ | Full hero, features grid, pricing section, hiring calculator, footer |
| Sign-up (`/sign-up`) | ✅ | Clerk-hosted, Google OAuth + email/password, first/last/email/password fields |
| Sign-in (`/sign-in`) | ✅ | Same Clerk flow with password recovery link |

| Issue | Severity | Notes |
|-------|----------|-------|

| No free trial tier | MEDIUM | All plans require immediate payment — no way for users to try before buying. "Start Free Trial" buttons redirect to sign-up, then to checkout |

## 6. NEW FEATURES TO SPEC — TOP 5 FOR REVENUE & DIFFERENTIATION

Ranked by revenue impact ÷ implementation effort:

### P0: Deliver Missing Pricing Features (2-3 sessions)
**The single highest-impact move.** You are actively **marketing 3 features that don't exist**. Fixing this:
1. **Custom branding** (Pro upsell) — DB migration for logo/colors + R2 upload + settings UI + inject into PDF/Word renders
2. **Audit log & version history** (Team differentiator) — New DB tables + version picker UI + audit log listing
3. **Auto-template updates** or rebrand to "Legislative tracker"

**Revenue impact:** Removes churn risk for Pro/Team buyers. Unlocks honest upsell path.
**Effort:** Session 1 (branding) + Session 2 (audit log) + Session 3 (polish)

### P1: Delete/Archive UI + Pagination (1 session)
- Add archive button to document list + detail page
- Add "Show archived" filter toggle + unarchive
- Add pagination (`limit`/`offset`) to GET `/api/documents`
- Add page controls to DocumentList

**Revenue impact:** Reduces UX friction. Essential before scaling beyond 50 docs/tenant.
**Effort:** 1 session (backend + frontend together)

### P2: Free Trial Tier (1 session)
- Add a free tier: 2 documents, no multi-user, no Word export
- No Stripe card required
- Post-sign-up: redirect to dashboard with trial banner

**Revenue impact:** **Highest conversion lever.** Every SaaS competitor has a free trial. Current "Start Free Trial" buttons go straight to payment — this is a conversion blocker.
**Effort:** 1 session (essentially just removing the Stripe gate for the first 2 docs and showing a CTA to upgrade)

### P3: Guided Onboarding Wizard (1-2 sessions)
- Post-sign-up check: first visit → show 3-step quick tour
- "Create your first document" CTA with hand-holding
- Tooltip highlights on key UI elements
- Progress bar showing "Your compliance setup: 0/5 documents"

**Revenue impact:** Converts trial users to paid. The first-document experience is the moment of truth.
**Effort:** 1-2 sessions (Zustand state for tour progress + onboarding component + tooltip system)

### P4: Payment History + Invoice View (0.5 session)
- Add Stripe Invoices API call to billing page
- List recent invoices with amounts and download links

**Revenue impact:** Low direct revenue but reduces billing support inquiries.
**Effort:** 0.5 session (Stripe already connected — just a new API call + table component)

## 7. RECOMMENDATION

**Do not build new features first.** The single highest-leverage action is to **deliver what you've already marketed** — custom branding, audit log, and either implement or rename the "legislative auto-updates" feature. Every day those appear on your pricing page without existing is a day you're misrepresenting the product to potential customers and creating churn risk for existing ones.

**Priority order:**
1. Fix pricing misrepresentation (custom branding + audit log) — 2-3 sessions
2. Add delete/archive UI + pagination — 1 session
3. Free trial tier — 1 session (can overlap with #1)
4. Onboarding wizard — 1-2 sessions
5. Billing niceties — 0.5 session
